문서의 구조와 슈퍼스타들 2월 20일

정보를 잘 정리정돈하기 위한 체계로 WEB 문서를 구조화하는 태그들이 있음
웹브라우저의 탭에 문서에 대해 설명하는 제목 태그
< title > 제목 < / title >
-> 검색엔진이 웹페이지를 분석할때 가장 중요하게 생각

문서에 영어가 아닌 문자를 사용했을때 글자가 깨지는 경우가 있음.
즉 웹페이지의 문자 표현방식과 웹브라우저의 문자 해석방식이 불일치 할때 발생함.
< meta charset="UTF-8" >
charset: 캐릭터세트, UTF-8: 문자 약속 기준

위의 두 태그는 본문이 아니라 본문을 설명하는 정보로 이를 head 태그로 묶어줌.
< head >
< title > 제목 < / title >
< meta charset="UTF-8" >
< / head >

본문은 body 태그로 구분함.
< body >
본문 내용
< / body >

head와 body를 감싸는 html 태그로 약속, 그리고 HTML로 제작되었다는 코드 추가
< !doctype html >
< html >
< head >
본문 설명
< / head >
< body >
본문 내용
< / body >
< / html >

다른 웹페이지들의 구조 확인해볼 것.
구조를 알았으니 시야가 넓어지고 예전에 보이지 않았던 것들이 보일 것이다.
